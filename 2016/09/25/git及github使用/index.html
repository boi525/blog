一、Git

1.什么是Git

  • 分布式的源代码管理工具
  • 在一个项目中，凡是有开发人员编写的代码都是都算是源代码
  • GIT是Linux之父（Linus Torvalds，被誉为活着的传奇）为了维护管理Linux源码写的一个工具
  • Git之前很多使用svn，vss，tfs，hs。。。

2.Git安装

安装相关的Git的客户端（命令行的）
安装界面化的Git客户端SourceTree（tortoisegit，smartgit）

3.Git命令速查

git init
git add .
git commit -m “first commit”
git remote add origin https://github.com/boi525/fdfdfd.git
git push -u origin master

获得密钥

email是你注册在github上的邮箱

4.Git简单使用

4.1 使用git init初始化一个本地仓库
会在本地文件夹中添加一个.git文件夹，用于记录项目所有的变更信息。

4.2 git status
用于查看本地仓储的变更状态。第一次查看，会显示未被跟踪的文件。
-s参数: 输出简要的变更日志。

4.3 git add .
将没有被跟踪的文件添加到跟踪列表(相当于git add –all)

4.4 本地忽略清单
在.gitignore文件中添加相应的忽略清单

4.5 提交
Git commit: 将本地的变化提交到本地的仓库文件夹归档
Git commit -m “说明”（必须有提交日志）

4.6 Git config
git config –global user.email “you@example.com”
git config –global user.name “Your Name”

4.7. Git diff
比较当前文件和文件的状态

4.8. Git log
git log -p

4.9. Git reset –hard “哈希值”
将代码回归到此处

总结:

  • Git不需要服务器，但是都需要，因为需要协同开发。
  • Git服务器，用来同步本地的代码仓库
  • Git是一个工具，github是一个网站（提供了远程仓库的功能）
  • Github网站提供的就是git服务（免费的服务，但是你提交的代码必须是开源的。）
